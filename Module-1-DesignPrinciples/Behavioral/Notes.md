## Observer Pattern 
```
->One obj triggers multiple objects 
->loosely couples from subject and observers
->Event management /Notifications
->register unregister dynamically
->system doesnt care who and all are there -> just notify's everyone
AUtomatically updates for all 

EVENT LISTENERS - NOTIFICATIONS - UPDATES 
```
## Stratergy Pattern
```
->defines ways to do the same task in diff way 
->chose on run time
->Can add new Stratergies
->Hides working
->each stratergy is independent and connected by stategy manager

PAYMENT METHODS - SORTING METHODS - DISCOUNT/MEMBERSHIP PRICE CALCULATIONS - FILE COMPRESSION
```
## Command Pattern 
```
->connects requests to object then does the task
->can be used to queue,undo and redo as the task are bound with the object
->Hides working
->easy to add new 
->Keeps track of commands 

REMOTE CONTROL - TEXT EDITOR - JOB QUEUE - ORDER SYSTEM
```
