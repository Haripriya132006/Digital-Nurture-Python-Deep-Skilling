## S-Single responsiblity 
-> one class must do one task
->For modularity and redability

## O-Open/Close Principle
->class must be open to extension close to editing
->makes sure the class's code aint changed after being written

## L-Liskov Substitution Principle
->A child must be able to preserve the behaviour of parent and not be different from it
->if the child doesnt the code needs to reevaluated

## I-Interface Segregation Principle
->one interface must do one job 
->modularity in interfaces

## D - dependency inversion Principle
-> classes should not be dependent on direct classes and must be dependent on abstractions/interfaces
-> provides flexiblity
