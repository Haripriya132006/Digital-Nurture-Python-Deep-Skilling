## Facade
-> Acts as a simplified, unified interface that hides the 
          complexities of a much larger subsystem.
-> Acts as an easy-to-use "front desk" or entry point so clients don't have to 
          navigate a maze of messy, complex classes.

SYSTEM SIMPLIFICATION - API DESIGN - HIDING COMPLEXITY

## Bridge
-> Decouples an object's abstraction (how it is controlled) 
          from its implementation (how it works) so both can evolve independently.
-> Replaces a massive, complex multi-level inheritance tree with two separate, 
          interconnected class hierarchies.

INDEPENDENT HIERARCHIES - REDUCING CLASS EXPLOSION - CROSS-PLATFORM DEVELOPMENT

## Composite
-> Lets you compose objects into tree structures to represent part-whole hierarchies.
-> Allows clients to treat individual objects and groups/compositions of objects uniformly.

TREE STRUCTURES - HIERARCHICAL DATA - FILE SYSTEMS & MENUS

## Flyweight
-> Minimizes memory usage by sharing as much common data as possible with other similar objects.
-> Separates an object's intrinsic state (shared/common) from its extrinsic state (unique/contextual).

MEMORY OPTIMIZATION - RENDERING MILLIONS OF OBJECTS - CACHING
